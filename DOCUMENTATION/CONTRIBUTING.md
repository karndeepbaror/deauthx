We welcome contributions from security researchers and protocol developers.

## Development Workflow
1. **Fork** the repository.
2. Create a **Feature Branch** (`git checkout -b feature/NewAttackVector`).
3. Implement and test in a **controlled environment**.
4. Submit a **Pull Request** with a detailed analysis of changes.

## Coding Standards
- Implement PEP 8 compliance.
- All new injection methods must include a "Safety Check" to prevent accidental disruption.
- Documentation updates are mandatory for new features.
